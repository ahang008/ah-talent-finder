# 设计哲学：为什么这个 Skill 是这样设计的

## 我的故事

我叫阿杭，00后，哈尔滨人，现居南京。风景园林专业，2024年毕业，进了央企干工地。

干了三个月就发现不对劲。每天填表格、跑现场、跟工人扯皮，我不知道自己能做什么，只知道不想做这个。那种感觉不是"工作不好"，而是"我跟这个环境之间隔着一层什么东西，说不清楚，但每天都在提醒我——你不属于这里"。

后来我用AI做了一次深度对话。聊我的家庭、聊我害怕什么、聊我做成的蠢事、聊我最骄傲的时刻。AI没有judge我，它只是记住了我说的每一句话，然后帮我看到了我自己看不到的东西——我最享受的是"把复杂的事情讲清楚"，我最讨厌的不是"工作"，而是"重复、应酬、没有成长"。我真正需要的不是"稳定"，而是"时间自主权 + 能力被看见 + 每一步可控有反馈"的组合。

AI帮我把这些翻译成了职业方向：产品经理。我试了，做了AI产品经理。但很快发现产品经理要依赖团队、依赖公司决策，很难直接助力自己创业。于是直接开始做自媒体。现在靠AI和自媒体养活自己，全职自由职业。

但这套方法我没自己藏着。过去三四个月，我把它做成了一套系统，给十多个朋友用了——有人定位完成了万粉博主，有人从建筑转行进了腾讯AIGC团队，有人从园林转成了自由职业接了私单。每个人都找到了自己的方向。

**知识确实是越公开越值钱的。** 你越让人听不懂，越让人觉得你没东西。所以我打算开源——不是开源一个工具，是开源一套帮你找到自己的方法论。

## 核心设计哲学

### 1. 精准名词 = 精准调用

AI不是真的"理解"你在说什么。它通过向量嵌入把文字转成高维空间里的位置，然后通过注意力机制匹配相关模式。

这意味着什么？**你对AI说的每一个词，决定了它能调动哪一块能力。**

所以这套系统里我大量使用心理学领域的精准术语——不是装逼，是因为我知道：

- 说"心流"（Flow），AI会调动契克森米哈赖的整套理论框架来分析你的注意力模式
- 说"阴影"（Shadow），AI会调动荣格的个体化理论来翻你的缺点
- 说"IKIGAI"，AI会调动四个圈交集的人生哲学来交叉验证你的职业方向

普通人跟AI聊天问"我适合做什么"，AI给你泛泛而谈。但如果你能精准地说"用加德纳多元智能分析我的天赋组合，用霍兰德RIASEC判断我的职业类型，用盖洛普34才干识别我的行为模式"——AI调用的计算资源、分析深度、结论质量，完全是两个量级。

**这叫"精准名词策略"：不是为了装专业，是为了精准调用AI最强的那部分能力。**

这套系统里每一个专业名词都是有意为之。9个理论框架（多元智能、VIA性格优势、霍兰德、盖洛普、心流理论、荣格阴影、IKIGAI、叙事心理学、依恋理论），不是堆砌，是9把不同的手术刀。每把对应一种分析维度，合在一起才能做一个完整的解剖。

### 2. 上下文窗口是硬约束

所有大模型都有上下文窗口限制。Claude再强，聊到一定长度也会丢失早期的信息。

这就是为什么这个系统不是"一次聊完"的。我把它拆成了**8个节点的分步对话**：

- 每个节点只收集一类信息，收集完立刻存档
- 存档不在AI脑子里，在用户本地文件系统里
- AI每次只处理当前节点的任务，不需要同时记住所有信息
- 到分析阶段（节点8、Step2节点3、Step3），再从存档里把所有字段读回来做综合分析

这解决了一个核心矛盾：**你需要AI做深度分析，但深度分析需要大量信息，而大量信息会撑爆上下文窗口。** 解法不是让AI记住一切，而是让AI"分段收集 + 本地持久化 + 按需回读"。

就像你做一道复杂的菜——不是把所有食材同时堆在案板上，而是切好一样放一边，最后一起下锅。

### 3. Python 管机械，AI 管思考

v1版本里，大量的token消耗在了"现在写到哪个文件、session状态怎么更新、文件夹建了没有"这种机械操作上。AI不擅长这个，也不该花token在这个上。

v2版本我把所有机械操作抽成了一个Python脚本（`scripts/session.py`）：

- 会话状态管理：JSON序列化，精确追踪当前节点、已完成节点
- 文件读写：自动路由"Step1-节点3"写到`01-轻量画像.md`
- 进度推进：自动标记上一个节点为已完成
- 断点续跑：用户随时退出，下次回来从断点继续

AI只需要关心一件事：**跟用户对话，并从对话里挖出insight。** 剩下的全部交给Python。

这背后是一个更大的设计原则：**AI是思考引擎，不是操作系统。** 文件IO、状态管理、流程控制——这些是传统计算机科学已经解决了的问题，没必要让AI再花token去思考。

### 4. 不是测评，是对话

市面上所有职业测评工具都有一个致命问题：**你填的是你以为的自己。**

- "你善于团队合作吗？"——人人都会打勾
- "你是否有创造力？"——你很难选"否"
- "你喜欢跟人打交道还是跟数据打交道？"——这是抽象问题，很少有人能准确回答

测评量表收集的是你的自我认知。但**你最大的天赋往往不在你的自我认知里**——它在你的行为模式里，在你的故事里，在你觉得"这有什么好说的"那些日常里。

所以这套系统不做选择题，不做量表，不做打分。**它让你讲故事。**

- 不讲"我善于什么"，讲"最近半年有没有一件事是你做成了、挺满意的"
- 不讲"我不喜欢什么"，讲"说一件最近让你特别烦的事"
- 不讲"我的性格特点"，讲"别人怎么说你，有没有哪句话你听了很多次"

然后AI从你的故事里找规律。你在不同场景下反复出现的决策模式，就是你的天赋。你自己都没意识到的行为模式，就是你的内心原密码。

**故事不会说谎。抽象结论会。**

### 5. 证据绑定：每个结论都要有出处

AI最大的问题是"听起来都对，但不知道从哪来的"。这套系统有一个硬约束：**每个结论都必须绑定用户说过的具体故事。**

- "你的天赋是'把复杂事情讲清楚'"→ 证据是你说的XX故事，当时你做了XX，结果XX
- "你的核心需求是'可控的确定性'"→ 证据是你三次提到讨厌不可控的场景，你做的所有尝试都是低风险验证

这不是为了严谨而严谨。是因为：

1. **用户需要相信这些结论。** 如果AI说"你适合做产品经理"但没有证据，用户只会觉得"哦"。如果AI说"你适合做产品经理，因为你三次提到自己喜欢把乱的事情理清楚、你在XX项目里主动做了需求整理、你朋友也说你是'人形说明书'"——用户会觉得"卧槽，对"。

2. **证据绑定本身就是质量保障。** 如果一个结论找不到证据，就不该输出。这倒逼AI在信息不足时继续追问，而不是用漂亮的空话糊弄过去。

### 6. 两套独立系统交叉验证

轻量版画像（Step1）准确度只有60%。这是因为只看表面信息——日常、能力、好恶——能做初步判断，但看不到深层模式。

深度版（Step2）通过原生家庭、成长关键事件、阴影/缺点、渴望/恐惧这些维度，把准确度拉到90%。

但最有意思的设计是Step4——命理交叉验证。

八字命理和现代心理学是两套完全独立的系统，互不认识。但从不同路径出发，如果能撞到同一个结论，这个结论就更稳。**心理学从你的真实经历里找规律，命理从你的出生时间推导先天倾向——两套系统互相参照，一致的增强信心，不一致的提供新视角。**

这叫"多模态验证"：不是让一个模型更准，而是让两个互不认识的模型互相校对。

## 技术架构的几个关键决策

### 为什么用 Markdown 而不是 JSON 存对话

存档文件（01-07）是Markdown格式，不是JSON。原因很简单：**这些文件用户要读。**

JSON是给机器读的，Markdown是在Obsidian里打开的。用户做完天赋发现后，这些文件不是一次性读物——三个月后、一年后回来看，会有新的理解。Markdown让它们成为用户知识库的一部分，而不是一个黑箱数据库。

### 为什么用 JSON 存会话状态而不是 Markdown

会话状态（`.session-state.json`）相反，用了JSON。因为：

- 会话状态不需要人类读，只需要Python精确解析
- JSON没有"格式不一致"的问题——Markdown如果用户手动改了，就解析不了
- 进度追踪、节点跳转、数据字段增删，JSON操作成本远低于Markdown解析

**人读的东西用Markdown，程序读的东西用JSON——各取所需。**

### 为什么开场脚本要先自我介绍

很多人觉得AI工具的"我是谁"部分是废话。但在这套系统里，开场脚本是有明确功能的：

1. **建立信任。** "我用同样的方法找到了自己的方向"——这句话让用户知道：你不是在跟一个瞎聊的AI对话，你是在跟一个"走过这条路的人"对话。
2. **设定预期。** "这不是测评、不是问卷、不是算命——就是我们俩聊天"——用户在30秒内就知道这不是又一个MBTI。
3. **降低防御。** AI先把自己的糗事说出来（"进了国企干工地，三个月发现不对劲"），用户在后续聊自己的缺点、恐惧时，防御心会低很多。

**开场不是寒暄，是心理安全的预设。**

### 为什么每个缺点都要立刻重构

Step2节点2里有一个关键规则：用户每说一个缺点，AI必须立刻正向重构。

这是故意的，不是为了"让用户感觉好"。

荣格心理学里有一个核心概念：**阴影（Shadow）里藏着最大的生命力。** 一个人最排斥的特质，翻转过来往往是他最深的天赋。比如：

- "我爱撒谎" → "你擅长在不同场景下调整表达方式，这是营销和公关的核心能力"
- "我做事慢" → "你对品质有极致要求，这在需要精准输出的领域是稀缺能力"
- "我社恐" → "你不需要社交来获取能量，你有独立的思考深度"

但在对话过程中，用户说自己的缺点时会进入一个脆弱状态。如果不立刻重构，用户会停止继续暴露——"我都说了两个了，还在说我不好，算了不说了"。

**即时重构不是安慰，是保持对话通道打开的机制。** 用户每暴露一个，你立刻翻成力量，他才敢继续暴露下一个。3-5个缺点收集完，才能在分析阶段做完整的阴影翻转。

## 大道至简

做了三四个月，迭代了十多个人的反馈，最大的感悟是什么？

**不是系统多复杂，是越简单越有效。**

- 不套框架，让选题决定用什么结构
- 不堆术语，用大白话说结论
- 不写空话，每个行动具体到"打开什么平台、搜什么、找谁、说什么"
- 不做测评，就聊天——因为聊天是人类最自然的信息交换方式

钩子、框架、结构这些，都是术。术是武器，不是每场的必选装备。真正的道是：**帮一个人看清楚自己，然后告诉他下一步具体做什么。**
