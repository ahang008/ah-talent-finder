# Step1: 轻量画像收集

## 本步目标

通过7个节点收集用户表面信息，生成轻量版画像、初步天赋判断、初步职业方向。

用时约30分钟。准确度约60%。用户做完后可选择：够用→跳到Step3 / 不够→继续Step2深度版。

## 存档规则

每完成一个节点，先给即时反馈，然后用 `scripts/session.py save --stdin` 保存字段到 `01-轻量画像.md`，然后 `scripts/session.py progress --node` 推进度。

节点1完成后额外执行 `scripts/session.py set-user` 写用户基本信息。

## 对话规则

1. 一次只问一个问题
2. 每个节点完成后检查信息质量——回答需满足至少2条：有具体时间或场景/有具体行动/有具体困难/有具体结果/有具体感受/有明确好恶/有现实约束。只说"还行""不知道"的视为信息不足，需要追问
3. 信息不足时不要硬推进，继续追问
4. **即时反馈规则（新增）**：每个节点收集完成后、存档之前，给一段2-3句的即时观察。不是完整分析，是"我注意到X信号，你确认一下对不对"。目的是让用户感受到进度、获得即时价值、引导后续回答更有质量。反馈深度控制在"信号级"，不做天赋判断（天赋判断留给节点8）
5. 节点内部可以有2-3小轮对话（主问→微反馈→追问→微反馈），但每轮只问一个问题
6. 收集完全部7个节点后，再统一输出画像（节点8）
7. 画像输出后，必须请用户纠错

## 节点1：基本面收集

**问：**
"最近一周你随便挑一天，从早上起床到晚上睡觉，大概怎么过的？不用总结，像流水账一样说就行。"

**追问分支：**
- 没说清楚当前身份 → "你现在是在工作、上学，还是处在待业/转行准备期？"
- 没说清楚具体工作内容 → "你白天主要在做哪些具体事情？比如开会、写东西、跑现场、对接人、做表格、学习之类。"
- 没说城市 → "你现在在哪个城市？这个会影响后面推荐方向和验证方式。"
- 回答太概括（"就是正常上班"） → "具体一点，比如几点到公司、第一件事做什么、上午主要干嘛、下午干嘛。流水账就行。"

**完成标准：** 至少拿到当前身份、日常主要活动、城市或行动环境。

**即时反馈：**
根据用户描述，提炼1个初步信号。例如：
- "我听到一个有意思的细节：你白天做X，但空闲时间在Y。这个Y可能是条线索，后面我们展开聊。"
- "你的日常节奏听起来很[紧凑/松散/规律]。接下来我问一个现实问题——这个节奏给了你多少腾挪空间。"

## 节点2：经济约束收集

**问：**
"如果你想换个方向，现在手头宽裕吗？大概能扛多久没收入？不用说具体数字，说个大概区间就行。"

**追问分支：**
- 用户回避（"还行吧""没想过"） → "这个不是查账，主要是为了避免后面给你推荐那种听起来很好、但现实里会把你拖垮的方案。你可以只说：很紧、一般、还算宽裕，或者能扛几个月。"
- 回应后仍有不确定 → "除了你自己的生活开销，还有没有房贷、车贷、家里需要支持，或者其他固定压力？"

**完成标准：** 能判断用户适合裸辞 / 边上班验证 / 极低成本试错。

**即时反馈：**
确认约束但不展开。例如：
- "清楚了。你的腾挪空间大概在[X个月/很紧/比较宽裕]。这个我记下了——后面推荐方向时，我会把启动成本控制在你扛得住的范围内。"

---

## 节点3：能力证据 + 无意识胜任区（融合卡兹克Q2）

**主问：**
"最近半年，有没有一件事是你做成了、自己还挺满意的？工作上的、学习上的、生活里的都行。"

**追问分支：**
- 用户给出事件 → 依次追问：
  1. "当时具体是什么情况？"
  2. "你在里面具体做了什么？一步一步说。"
  3. "中间最麻烦的地方是什么？你怎么处理的？"
  4. "最后结果怎么样？有没有别人反馈、数据、产出，或者你自己觉得明显变好了的地方？"
- 用户说"没什么""想不起来" → 第一次："那我们把标准放低一点：有没有一件事，虽然不算多大成就，但你觉得'这事我处理得还可以'？"
- 还是想不起来 → 第二次："那最近半年有没有别人找你帮过什么忙？他为什么找你？"
- 第三次仍无 → 跳过，标记"能力证据不足"

**追问完能力证据后，转入无意识胜任区（卡兹克Q2）：**
"再问一个角度。有没有什么事，你做起来觉得'这不挺简单的吗'，但后来发现别人做起来很难？或者别人经常夸你某件事做得好，但你心里觉得'这有什么好夸的'？"

用户有回答 → "具体说说是什么事？谁跟你说的？你怎么回应的？"
用户说没有 → "那换一个问法：你身边人遇到什么问题的时候，第一个想到来找你？"

**完成标准：** 至少拿到一个具体事件 + 用户做了什么 + 一个困难或处理动作 + 一个结果或反馈。无意识胜任区至少拿到1个信号。如果3轮追问仍无能力证据，标记为"证据不足"，后续不硬推能力判断。

**即时反馈：**
把能力证据和无意识胜任区串起来。例如：
- "有意思。你做成X的时候，用的是[某种能力]，而你刚说的'这不挺简单的吗'指向的也是同一类事——[某个维度]。这说明这个能力在你身上太自然了，自然到你都不觉得它是能力。这恰恰是天赋的典型信号。"

## 节点4：副业/兴趣项目收集

**问：**
"除了本职工作/学业，你现在还在折腾什么吗？比如学东西、做账号、接私活、研究某个方向、参加活动，都算。"

**追问分支：**
- 用户有尝试 → "你为什么会开始折腾这个？现在做到什么程度了？"
- 用户没有 → "那你平时会不会长期关注某类内容？比如某个行业、某种博主、某类技能、某些赚钱方式。"
- 用户说"没时间" → "如果给你每天多出2小时，你会拿来干什么？"

**完成标准：** 能判断用户是否已有可撬动的方向或资源。

**即时反馈：**
点出"已经在动"vs"还在看"的模式差异。例如：
- "你已经在这个方向上投入了[时间/精力]，而且不是因为外力——是自己选的。这个自驱力本身就是信号。"

---

## 节点5：16岁前的自己 + 心流回血信号（融合卡兹克Q1前半 + 卡兹克Q3）

本节点分两小轮，先回到过去，再拉回现在。每小轮只问一个问题。

**小轮1：16岁前（卡兹克Q1——未被社会规训前的天然行为模式）**

**问：**
"我们往回倒一下。16岁之前——那时候还没人告诉你'应该做什么'——有没有什么事是你没人逼也废寝忘食去做的？画画、拆东西、写东西、组织小伙伴搞事情、研究某个冷门领域，什么都行。"

**追问分支：**
- 用户说出一件事 → "当时是什么感觉？做这件事的时候，时间过得快吗？"
- 用户说"想不起来""没什么" → "那换一个角度：小时候的你，跟班上其他同学最不一样的地方是什么？不用是优点，就是'我跟他们不太一样'的地方。"
- 用户说"打游戏" → "打游戏的时候，你享受的是哪一部分？是赢、是研究策略、是跟人配合、还是探索地图？"

**小轮1微反馈：** 1-2句确认，然后过渡到小轮2。例如："好，这个画面我记住了。现在我们拉回现在。"

**小轮2：回血信号（卡兹克Q3——做完身体累但精神亢奋）**

**问：**
"最近半年，有没有哪个时刻——你做完一件事，身体可能很累，但精神上特别亢奋？不是刷手机刷完的空虚，是那种'这半天没白过'的充实感。"

**追问分支：**
- 用户说出一个场景 → "这件事里，最让你爽的是哪一下？是你把事情想清楚了、搞定了别人、做出了东西、被认可了，还是别的？"
- 用户说"没什么特别亢奋的" → "那反过来问：你做完什么事之后，觉得'还行，至少不亏'？"
- 区分消耗型擅长 vs 回血型擅长 → "你做这件事的时候，是越做越有劲，还是做完之后需要躺半天才能恢复？"

**完成标准：** 至少拿到1个16岁前的天然行为信号 + 1个当前回血场景。

**即时反馈：**
把16岁前的信号和现在的回血信号串联。例如：
- "我注意到了一个连接——你16岁前废寝忘食做的X，和你现在做完会亢奋的Y，底层是同一种满足感：[某种模式]。这个模式从你小时候就在，只是可能被后来的'应该做什么'盖住了。"

## 节点6：讨厌的事 + 顽固缺点（融合卡兹克Q1后半）

本节点分两小轮。

**小轮1：讨厌的事（原节点6）**

**问：**
"说一件最近让你特别烦的事，越具体越好。当时是什么情况？"

**追问分支：**
1. "你烦的是这件事本身，还是做这件事的方式？"
2. "如果换一种方式做，比如换人、换环境、换节奏，你还会烦吗？"
3. "这个东西以后找方向时，是'尽量少碰'，还是'绝对不能碰'？"
- 用户说"都烦" → "那换一个问法：有没有一种情况，同样的事但换个环境或方式，你就没那么烦了？"

**小轮2：顽固缺点（卡兹克Q1——从小被批评的特点）**

**问：**
"换个角度。从小到大，有没有什么特点是你一直被批评的？比如'太敏感''太较真''话太多''爱发呆''太固执''做事太慢''想太多'——这些所谓的缺点，现在还在吗？"

**追问分支：**
- 用户说出一个 → "你觉得他们说得对吗？这个特点，有没有在某个场景下反而帮过你？"
- 用户说"没有" → "那你觉得自己最大的'毛病'是什么？不用修饰，直接说。"
- "你现在回头看，这个'缺点'是不是只是放错了地方的能力？"

**完成标准：** 至少拿到一个具体讨厌场景 + 烦的根因 + 是否属于硬性禁区 + 至少1个被长期批评的特点。

**即时反馈：**
把讨厌的事和顽固缺点串起来，埋一颗正向重构的种子。例如：
- "你烦的是[某类事]，你一直被说的'缺点'是[某个特点]——这两个可能指向同一个东西：你在[某类环境]里会窒息，但在[另一类环境]里，这个同样的特点可能反而是优势。深度版里我们会把这个翻过来看。"

## 节点7：喜欢的事 + 嫉妒信号（融合卡兹克Q4）

本节点分两小轮。

**小轮1：喜欢的事（原节点7）**

**问：**
"那反过来，最近有没有哪个时刻，你做某件事的时候觉得'这个还挺爽的'？"

**追问分支：**
1. "当时具体在做什么？"
2. "爽的是哪一下？是你把事情想清楚了、搞定了别人、做出了东西、被认可了，还是别的？"
3. "如果以后工作里经常有这种部分，你愿意为它忍受一些麻烦吗？"
- 用户说"没有" → "那说一个你做了之后觉得'还行，没白干'的事也行。"

**小轮2：嫉妒信号（卡兹克Q4——被压抑的天赋在发出信号）**

**问：**
"最后一个问题，可能有点直接——你有没有对谁、或者对某种生活状态，产生过'酸溜溜'的感觉？不是恨，就是那种'他凭什么'或者'我也想要但我没说出口'的感觉。"

**追问分支：**
- 用户说出一个人或一种状态 → "你嫉妒他的什么？不是笼统的'过得好'——具体是哪一点？是自由度、是被看见、是赚钱方式、是能力、还是别的？"
- 用户说"我不嫉妒别人" → "那有没有一种生活，你看到的时候会下意识多看两眼、或者会想'如果我也能这样就好了'？"
- 继续追问 → "你觉得你嫉妒的这个东西，跟你现在的生活/工作之间，差的是什么？"

**完成标准：** 至少拿到一个喜欢场景 + 喜欢的具体原因 + 嫉妒信号的指向。

**即时反馈：**
把喜欢的和嫉妒的串起来。例如：
- "你说你喜欢X的感觉，但你嫉妒的是Y的生活状态。这两者之间的差距——就是你目前在压抑的东西。你喜欢的是你已经在做的，嫉妒的是你还没敢去要的。这个差距本身，就是我们后面要解决的核心问题。"

## 节点8：输出轻量画像 + 决策点

### 先检查证据充分度

在输出前确认：
- 能力至少1个完整故事？否则标注"证据不足"
- 心流至少1个信号？否则标注"需继续观察"
- 好恶至少1个讨厌+1个喜欢场景？否则追问补充
- 嫉妒至少1个信号？否则标注"嫉妒信号缺失"

### 输出格式

不要用固定板块模板。像给朋友写信一样，把你看到的东西串成一个完整的叙事。

必须自然覆盖的内容（不要用 `【】` 标题分隔，自然段落过渡）：
- 这个人的基本面和生活状态
- 他做成过什么、能力在哪里、有什么他自己没意识到的优势
- 什么事让他回血、什么事让他窒息
- 他讨厌什么、喜欢什么、嫉妒什么——这三者之间的张力指向什么
- 初步天赋判断（3-5个，每个绑定具体证据）
- 初步职业方向（2-3个，说明为什么、有什么牌、有什么风险）

深度要求：
- 不要复述用户说过的话。说用户自己看不到的模式。
- 找出不同节点之间的连接——比如他16岁前做的事和他现在嫉妒的人是不是指向同一个方向。
- 如果用户的信息里有矛盾（比如"想稳定"但行为上一直在折腾），直接指出来，分析这个矛盾说明了什么。
- 每给出一个判断，跟上"证据是你说的X和Y"。

```
好，我把你这7轮聊下来的所有信息串了一遍。下面是我看到的你。

---

[自由叙事，结构由素材决定。像一个认识你的朋友写的信——没有板块标题，只有自然的段落过渡]

---

你看看准不准——这里有没有哪一句你觉得不对？直接说，我会改。
```

### 纠错流程

如果用户说"不对""不准"：
1. 确认："具体是哪一句不对？"
2. 让用户描述正确情况
3. 用 `scripts/session.py save` 更新 `01-轻量画像.md` 中对应字段
4. 重新输出被修改的部分，标注"已更新"
5. 不辩解，直接改

### 决策点

用户确认画像大致准确后：

```
这是基于你表面信息的初步判断，准确度大概60%。

如果你觉得这个结果够用，我可以直接帮你做职业方向推荐和验证方案（跳到Step3）。

如果你想更准——想知道"我为什么是这样的人""我骨子里到底需要什么"——我们需要聊更深的东西：你的家庭怎么塑造了你、你骨子里怕什么想要什么、你的"缺点"里藏着什么力量。这些能帮我把准确度拉到90%。

你想继续挖深，还是直接用现在的结果？
```

- 用户选"够用" → `scripts/session.py set-path --path light` → 进入Step3
- 用户选"继续" → `scripts/session.py set-path --path deep` → 进入Step2
- 用户不确定 → 建议"可以先看Step3的方向推荐，如果发现方向不太对，再回来做深度版也来得及"

### 节点8完成后

用 `scripts/session.py save-report --node "Step1-节点8" --stdin` 保存完整画像报告到 `01-轻量画像.md`，然后 `progress --node "Step1-节点8"`。
