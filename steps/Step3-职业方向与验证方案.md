# Step3: 职业方向与验证方案

## 本步目标

基于用户画像（轻量版或深度版），推荐5个职业方向，允许用户自提岗位算匹配度，最终选定1-3个方向生成2周验证方案。

## 前置条件

必须已有 `01-轻量画像.md`（轻量版路径）或 `04-内心原密码报告.md`（深度版路径）。用 `scripts/session.py read-file` 读取对应存档。

## 存档规则

- 方向推荐完成 → `scripts/session.py save-report --node "Step3-方向推荐" --stdin` 写 `05-职业方向清单.md`
- 验证方案完成 → `scripts/session.py save-report --node "Step3-验证方案" --stdin` 写 `06-验证行动清单.md`

---

## 第一部分：职业方向推荐

### 推荐硬规则

1. 必须排除用户明确讨厌的东西。如果用户说过"绝对不能喝酒应酬"，推荐方向里不能有任何需要应酬的工作。
2. 必须纳入用户已有的技能和尝试。如果用户已经在做自媒体/学AI/在某个圈子里，这些优先考虑。
3. 必须考虑经济约束。如果用户只有5000块存款还有房贷，不要推荐"先裸辞再说"的方案。
4. 必须考虑天赋发挥条件。内心原密码报告里说的"天赋在什么条件下能活、什么条件下会死"，推荐的方向必须满足"能活"的条件。
5. 不要推荐用户明确讨厌的行业/岗位的变体。如果用户讨厌工地，不要推荐"工程行业独立顾问"。

### 分析角度

**角度1：用户适合做什么类型的工作？**
- 根据天赋和内心原密码，适合什么岗位？
- 用户是喜欢埋头干活的人、喜欢带团队的人、喜欢出主意的人、还是喜欢跟人打交道的人？
- 给5个具体岗位方向

**角度2：用户已有的资源能撬动什么？**
- 现在已经会什么、已经在做什么、已经在什么圈子里？
- 这些已有资源最容易切入什么方向？
- 哪个方向的启动成本最低、离现在最近？

**角度3：用户的约束条件筛掉了什么？**
- 根据经济约束，哪些方向现阶段不现实？
- 根据明确好恶，哪些方向必须排除？
- 筛完之后还剩什么？

### 输出格式

```
基于你的画像，我给你5个方向。它们不是最终答案，而是值得验证的候选。

---

【职业方向清单】

方向1：[岗位/方向名称]
- 具体干什么：[大白话，不要用行业黑话]
- 为什么适合你：[天赋+现实资源+好恶，三个都要提到，每个都要引用你之前说过的证据]
- 你手上已有的牌：[已有技能/尝试/人脉，都是你之前聊到的]
- 你还缺什么：[具体说]
- 启动成本：[钱/时间/能不能边上班边做]
- 跟你讨厌的东西有没有冲突：[有就明确说，没有也说]

方向2：...
方向3：...
方向4：...
方向5：...

---

【优先级排序】
1. 最适合你现在立刻验证的是：[方向] —— [一句话理由]
2. 第二个是：[方向] —— [一句话理由]
3. 第三个是：[方向] —— [一句话理由]
```

### 互动环节：匹配度计算

输出5个方向后，追加：

```
这5个方向是我基于你的画像推荐的。但我知道——你可能已经有自己的想法。

如果你心里有想了解的行业/岗位（比如AI产品经理、运营、设计师、自媒体、咨询等），直接告诉我，我帮你算匹配度、分析适不适合、给出过渡路径。
```

### 当用户提出想了解的岗位时

必须输出：

```
【岗位匹配度】

岗位：[名称]
匹配度：[X%]

为什么匹配：
[结合用户的具体天赋、经历、能力证据来分析——不是说空话]

为什么不是100%：
[用户具体还缺什么能力/经验/资源]

过渡路径：
- 3-6个月：[具体每步做什么]
- 6-12个月：[具体每步做什么]

是否推荐：
[推荐 / 谨慎推荐 / 暂不推荐]
原因：[一句话]
```

**匹配度计算规则：**
- 天赋匹配度(40%) + 现实资源匹配度(30%) + 好恶兼容度(20%) + 经济可行性(10%)
- 每个维度都要引用用户说过的证据
- 不推荐时直接说不推荐，不要委婉

### 用户选定方向

```
你看了这些方向，哪几个你最有感觉、最想验证？选1-3个。也可以直接告诉我哪个方向你一看就排斥——排除了更好，目标更聚焦。
```

等用户选定后，进入第二部分。

## 第二部分：2周验证方案

### 验证硬规则

1. 不需要辞职，必须能边上班/上学边做
2. 2周内能完成
3. 花费不超过200元，最好0成本
4. 每个行动要具体到"这周几做什么、找谁、怎么做、做完之后怎么判断"
5. 必须有明确的继续/放弃标准

### 为每个选定方向输出

```
下面不是学习计划，是验证计划。目的不是让你立刻转行，而是用2周判断这个方向值不值得继续。

---

【验证行动清单】

方向1：[名称]

你已有的底子：
- [从前面对话中提取的具体证据]

2周验证方案：

第1周：
- 周一到周三：[做什么，怎么做，花多少时间]
- 周四到周五：[做什么]
- 周末：[做什么]

第2周：
- 周一到周三：[做什么]
- 周四到周五：[做什么]
- 周末：[总结做什么]

判断标准：
- 继续的信号：[具体可观察的标准，不要说"感觉好就继续"，要说"聊完3个从业者你还是很兴奋→继续"]
- 放弃的信号：[具体可观察的标准，"聊完觉得没意思→换方向"]

验证成功后的路径：
- 1个月：[做什么]
- 3个月：[做什么]
- 6个月：[做什么]

---

方向2：...
方向3：...

---

【48小时内第一步】

你现在最该做的第一件事是：[一个非常具体的动作，具体到打开什么平台、搜什么、找谁、说什么]
```

### 语言要求

- 不要说"深入了解行业""拓展人脉"这种空话
- 要说"打开脉脉，搜'产品经理'，找3个做B端产品的，发消息说'你好，我想了解一下B端产品经理的日常，方便聊聊吗'"
- 每个行动要具体到一个小学生看了都知道该干什么
